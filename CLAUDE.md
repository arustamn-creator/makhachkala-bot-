# CLAUDE.md

Этот файл содержит инструкции для Claude Code (claude.ai/code) при работе с данным репозиторием.

## Обзор проекта

Telegram-бот в одном файле для местного маркетплейса услуг по дому в Махачкале. Связывает заказчиков с мастерами (сантехниками, электриками и др.) и службами доставки. Весь пользовательский интерфейс на русском языке.

## Запуск бота

```bash
pip install pyTelegramBotAPI
BOT_TOKEN=<токен> python bot.py
```

Тесты, шаги сборки и CI/CD отсутствуют.

## Конфигурация

Три константы в начале `bot.py`:

| Переменная | Расположение | Назначение |
|---|---|---|
| `BOT_TOKEN` | `os.environ.get("BOT_TOKEN")` | Токен Telegram-бота — передаётся через переменную окружения |
| `ADMIN_ID` | жёстко задан, строка 17 | Telegram ID пользователя, которому приходят все заявки |
| `MINI_APP_URL` | жёстко задан, строка 16 | URL сопутствующего веб-приложения на Vercel |

## Архитектура

Весь бот находится в `bot.py` (~450 строк). Маршрутизация сообщений состоит из трёх уровней:

1. **Обработчики команд** (`/start`, `/help`, `/masters`, `/shops`, `/delivery`) — точки входа, которые делегируют вызов общим функциям `send_*`.
2. **Обработчик текста** (`@bot.message_handler(func=lambda m: True)`) — перехватывает нажатия кнопок `ReplyKeyboardMarkup`, проверяя вхождение метки кнопки в `message.text`. Передаёт управление функциям `send_*` или `bot.register_next_step_handler`.
3. **Обработчик callback-запросов** (`@bot.callback_query_handler`) — обрабатывает нажатия кнопок `InlineKeyboardMarkup` через строки `call.data`. Маршрутизирует по точному совпадению или префиксу (например, `apply_cat_plumber`).

### Поток обработки заявок

`receive_application` — единственный обработчик для всего текста, который вводит пользователь (заявки на услуги, заказы доставки, регистрация мастера). Он:
1. Подтверждает получение заявки пользователю.
2. Пересылает текст сообщения вместе с данными пользователя администратору на `ADMIN_ID` через `bot.send_message`.

Регистрируется динамически через `bot.register_next_step_handler` из трёх мест: кнопка «Оставить заявку», callback-кнопки `order_*` (доставка) и callback-кнопка `master_apply`.

### Данные контента

Категории мастеров и варианты доставки хранятся в обычных словарях внутри `callback_handler` (`CATEGORY_TEXTS`, `DELIVERY_TEXTS`) — без базы данных или внешней конфигурации. Чтобы добавить или изменить категорию, нужно отредактировать и словарь, и список кнопок в `send_masters`/`send_delivery`.
